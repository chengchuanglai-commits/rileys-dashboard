# Plan F — 分档出场策略 设计文档

**Goal:** 在 Plan D 参数基础上，将每笔交易拆成 2 档分批出场，最大化每次 AI 预测的价值。

**Architecture:** 新增 `backfill-portfolio-f.py` 回测脚本，输出 `portfolio_f.json` + `portfolio-f.js`，集成到 dashboard 对比面板和折线图。

**Tech Stack:** Python / yfinance / IBKR 佣金模型 / 同 B/C/D/E 方案

---

## 策略参数

| 参数 | 数值 | 来源 |
|------|------|------|
| 仓位 | $500 / 单笔 | 同 D |
| 最大持仓 | 3 个交易日 | D 基础上+1天，给 2 档更多时间 |
| 止损 | -3%（固定，全仓止损） | 同 D |
| 跳空过滤 | 不利跳空 >1% 跳过 | 同 D |
| 佣金 | IBKR $0.005/股 最低 $1.00/单 | 同其他方案 |

## 出场规则（核心差异）

```
总股数 = int(500 / entry_price)
第 1 档 = shares // 2       → 目标 TP1 = 入场 + (原始TP - 入场) × 50%
第 2 档 = shares - 第1档    → 目标 TP2 = 原始 TP（+15%）

出场优先级（每天按顺序检查）：
  1. SL 触达 → 全部剩余股数立即止损
  2. TP2 触达（且第1档已平）→ 第2档止盈
  3. TP1 触达（且第1档未平）→ 第1档止盈
  4. 第3天收盘 → 全部剩余股数到期平仓

佣金：
  入场：ibkr_commission(shares)（一次）
  第1档出场：ibkr_commission(第1档股数)
  第2档/止损/到期：ibkr_commission(第2档剩余股数)
  最多3笔佣金（vs 当前方案的2笔）
```

## 数据示例

```
SELL KLIC @ $47.20  原始TP $40.12  SL $48.62

总股数：10股（$500 ÷ $47.20 = 10）
第1档：5股，TP1 = $47.20 - ($47.20-$40.12)×0.5 = $43.66
第2档：5股，TP2 = $40.12
止损：$48.62（两档共用）
佣金：入 $1.00 + 出1 $1.00 + 出2 $1.00 = $3.00

情景1：TP1触达后TP2也触达
  第1档盈利：5×($47.20-$43.66) = $17.70
  第2档盈利：5×($47.20-$40.12) = $35.40
  总毛利：$53.10，扣佣金$3.00 → 净$50.10

情景2：TP1触达后反转至止损
  第1档盈利：$17.70，扣$2.00佣金 → +$15.70
  第2档亏损：5×($47.20-$48.62) = -$7.10，扣$1.00佣金 → -$8.10
  净：+$7.60（仍盈利，因第1档锁住了利润）

情景3：直接触发止损（从未达到TP1）
  全部亏损：10×($47.20-$48.62)×(-1) = -$14.20，扣$2.00 → -$16.20
  （与D方案相同，止损机制一致）
```

## 输出文件

- `scripts/backfill-portfolio-f.py` — 回测脚本
- `data/portfolio_f.json` — 持仓数据
- `dashboard/portfolio-f.js` — 前端数据
- `dashboard/index.html` — 新增 F 方案面板 + 折线图曲线

## 与现有方案的对比目标

Plan F 在以下情况下应优于 Plan D：
- 股票在 TP1 后继续涨到 TP2 的比例较高（>40%）
- 止损率不比 D 高（同样的 -3% SL）

Plan F 在以下情况下会差于 Plan D：
- 大多数交易都是直接触发止损或到期平仓（2 档优势无法发挥）
- 佣金多 $1/笔导致小盈利被吃掉

## GitHub Actions 集成

- `refresh-portfolio.yml`：盘中每 30 分钟刷新 F 方案
