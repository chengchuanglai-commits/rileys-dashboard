# 静态互补组合(对照基线)设计 spec

**日期**:2026-07-10
**状态**:brainstorming 定稿(Riley 授权"够好就执行",免 spec 复审)

## 背景与目标

Riley 的真实初衷:**不同市场环境跑不同的腿、组合收益最大化**,不是选定一条腿跑到黑,也不是只买指数。红队后确定:**"轮动"证据上会输(检测滞后,见 [[strategy_regime_playbook]]、legs_tested_summary 的 regime 段)**;能成的方向更可能是**低相关互补组合**。

本 spec 只做**第一步:静态加权组合(固定权重、只定期再平衡、不预测不轮动)当对照基线**。将来任何轮动方案必须先打赢它,否则轮动只是白加复杂度+过拟合面。

**目标**:检验"给指数核心加一条互补卫星(动量)"能否在**风险调整后(夏普)** 跑赢裸指数。

## 红队补丁(为什么这么设计)

1. **只活过一个 regime**(6周 paper)→ 用**各腿多年回测收益流(2021-now,复用 `analyze-leg-correlation.py` 已验证的策略模拟)**,不用 paper。
2. **相关性是噪音**(小样本)→ 权重用**固定启发式,不优化**(min-variance 会拟合噪音);做**稳健性带**(90/10~70/30),好处只在单点冒出=脆弱=不信。
3. **做空腿最负相关(−0.53)但是幻觉**(借券费/借不到,实测 −0.60 夏普、−56% 收益)→ **剔除做空**,只用 QQQ+动量。
4. **防御不双算**:lev 自带 200MA 闸门崩盘去现金,和动量"崩盘去现金"同向→假分散。故**锚用永远在场的 1x QQQ**,不用 lev。
5. **基准要诚实**:同时对照 ①裸 1x QQQ ②裸 lev 杠杆闸门指数(打不赢 lev = 答案可能是"直接做 lev")。

## 成分与方法

- **锚**:1x QQQ(永远在场)
- **卫星**:动量腿 MOM-MA(0.41 相关、崩盘去现金、可执行)
- **权重**:锚重固定带 = {90/10, 80/20, 70/30}(QQQ/动量),每日定权重再平衡(constant-weight)
- **窗口**:2021-06 → now(~5年,FMP 缓存)
- **实现**:扩展 `analyze-leg-correlation.py`(复用其 QQQ/动量净值曲线),新增 lev 基准(`lev_engine.simulate_leverage`,同窗口重归一)+ 权重带组合 + 对照表 + 裁决。纯研究,**不接实盘、不接自动化**。

## 成败标准

组合 CAGR/波动/夏普/最大回撤,对照裸 QQQ 与裸 lev:
- **过**:某锚重组合夏普稳健(三档一致)高于裸 QQQ → 互补有价值,值得继续做轮动层。
- **不过**:打不赢裸 QQQ → 诚实结论"我们没有能提升风险调整收益的互补腿",轮动方向应搁置或回到"指数(可能加杠杆)为主"。

## 诚实边界

5年是单一大牛市窗口;动量腿有幸存者偏差(绝对收益偏乐观,相关性/夏普相对结论较稳)。回测是强先验非圣旨,真裁决永远在前向。
